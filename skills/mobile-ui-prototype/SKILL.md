# Mobile UI Prototype

按用户提供的手机截图还原可交互 HTML 原型，优先做视觉对齐，不要先写规范文档。

## Use when
- 用户要“按截图还原界面”
- 用户要“可点击访问的 HTML 原型”
- 用户强调“不要规范文档，直接出页面”
- 用户要求做底部 Tab 多页面（首页/业务/经营/我的）

## Do not use when
- 只要文字方案，不要代码
- 已有完整前端工程且只改一个小组件
- 用户明确要求输出 Design Spec / Token 文档而不是页面

## Inputs needed
- 目标截图（至少 1 张；多页最好全给）
- 画板尺寸（默认 `390x844`）
- 页面清单（如：首页 / 业务 / 经营 / 我的）

## Workflow
1. 固定画板：`390x844`，不要放大字号造成比例失真
2. 先还原首页（蓝色头部、快捷入口、KPI、常用工具、趋势图、底栏）
3. 再还原其余 Tab 页，保持同一套视觉语言
4. 使用内嵌 SVG 图标，禁止 emoji 图标
5. 增加基础交互：Tab 切换、点击反馈、筛选切换
6. 输出单文件 HTML，可直接用浏览器打开

## Output format
- 产出文件：`jxc-mobile-prototype.html`（或用户指定名字）
- 回复只说明：
  - 文件路径
  - 包含页面
  - 已实现交互
  - 下一步可选微调项（像素级对齐）

## Quality checklist
- [ ] 是否固定 `390x844`
- [ ] 是否使用 SVG 图标（不用 emoji）
- [ ] 字号/间距是否接近截图，无明显放大
- [ ] Tab 切换是否可用
- [ ] 页面是否可直接本地打开

## Example prompts
- 按我截图做 4 个 Tab 的移动端 HTML 原型
- 不要文档，直接给我可点的界面
- 按 390x844 一比一还原首页
